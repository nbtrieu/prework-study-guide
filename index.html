HTML


  • Hypertext Markup Language (HTML): made up of elements which work together to provide content and the basic framework for all pages on the web.
  • The head element contains information, or metadata, that helps the browser to render the page correctly. Elements included within the head are not visible when a user looks at a webpage.
  • The body element contains the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.
  • Semantic HTML elements: their names describe their content and the role they play on a webpage.
  • BEST PRACTICE: Use semantic elements whenever possible, especially when the content is visible to users.
  • Within the body elements, The header appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • The main element appears in the middle of the page and specifies the main content of a document that is unique to the document.
  • The footer appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.
  • The section elements are semantic elements that define a section in a document that contains thematic content.


CSS


  • margin property: indicates space around the outside of an element.
  • padding property: adds space around the content inside an element.
  • * wild card symbol: signifies ALL elements on the page getting the same style properties.
  • display property with block value: assigns each image to start on a new line.


Git


  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds all modifications in the current working branch to the staging area.
  • git commit -m: creates a commit that captures a snapshot of the currently staged changes.
  • git pull origin main: ensures that we have the latest version of the project on our local computer before starting to work.
  • git push origin feature/starter-code: pushes the changes we have locally to our remote GitHub branch.
  • Then, on the GitHub repo: select the feature branch -> give the pull request (PR) a descriptive title -> designate the base branch as main and the comparison branch as feature/starter-code -> Select the "Create Pull Request" button -> request that one or two other team members review the code before it is merged into the base branch.


JavaScript


  • Variables are named containers that store data/values - can be declared by the keywords let, const, or var.
  • Control flow is the order in which a computer executes code in a script.
  • We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
  • Conditionals are code structures used to test if an expression returns true or not. A very common form of conditionals is the if...else statement.
  • A for loop uses the pattern of indices to run a task on all the items in an array by letting a code block to be executed over and over.
  • A function is different from a for loop or conditional statement because functions must be called by name in order to be executed.
  • Calling a function to be executed means controlling where in the control flow a task is executed or if at all.
